Connection

The code utilizes the Flask framework to create a web application that communicates with an Arduino board connected to the computer via a serial connection. The Flask application serves a web page called "index.html" and exposes an API endpoint ("/_stuff") for handling requests from JavaScript.

Controller 

The code captures video frames from the webcam using OpenCV's VideoCapture class. Each frame is processed to detect hand landmarks using Mediapipe's Hands module. The code then maps the detected landmarks to specific hand gestures by analyzing the relative positions of key landmarks.

Overview of the connection code

The code of the connection combines Flask, JavaScript, and the serial connection to create a web-based interface for controlling and monitoring an Arduino board. The web page allows users to send commands to the Arduino board and receive data updates in real-time, enabling interactive communication between the user interface and the physical system connected to the Arduino board.

Explanation of the Connection code

Importing Libraries and Initializing Serial Connection

The code starts by importing the necessary libraries: Flask, json, time, and serial.A serial connection is established with the Arduino board using the serial.Serial() function. The specified parameters are the serial port ("COM7") and the baud rate (9600). 

Flask Application Setup

An instance of the Flask application is created using Flask(__name__).

API Endpoint for Handling Requests

The @app.route('/_stuff', methods=['GET', 'POST']) decorator defines an API endpoint ("/_stuff") that can handle both GET and POST requests. When a POST request is received, the code extracts the speed and accident values from the request's JSON payload. The speed and accident values are concatenated into a command string in the format expected by the Arduino board. The command string is encoded and written to the serial connection using arduinodata.write(cmd).

Reading Data from Arduino

The code checks if there is any data available in the serial buffer using arduinodata.inWaiting(). If data is available, it reads a line of data from the serial connection using arduinodata.readline(). The received data is decoded from bytes to a string and stripped of any newline characters. The string is split into a list of values using the comma as the delimiter. The resulting list, representing the data packet received from the Arduino, is assigned to dataPacket

Returning JSON Response

The data packet (dataPacket) is converted into a JSON response using jsonify(). The JSON response is returned to the JavaScript code on the web page.

Serving the Web Page

The @app.route('/') decorator defines the root route ("/") of the Flask application. When a request is made to the root route, the code retrieves the paths of two images and converts them into a JSON string (image_paths_json). The web page "index.html" is rendered using the render_template() function, with the image paths JSON string passed as a parameter.

Running the Flask Application

The code runs the Flask application using app.run(). The application listens for incoming requests and handles them accordingly.

Explanation of the Controller code

Importing Libraries and Initializing Objectsn

The code starts by importing the necessary libraries: cv2 for computer vision tasks, mediapipe for hand gesture recognition, and pyautogui for simulating keyboard inputs. The line mpHands = mp.solutions.hands imports the Hands module from Mediapipe. An instance of the Hands class from the mpHands module is created and assigned to the variable hands. The max_num_hands parameter is set to 1 to detect a maximum of one hand, and min_detection_confidence is set to 0.7 to control the minimum confidence required for a hand detection.  

Real-time Gesture Recognition Loop

The code enters a while loop that continuously captures frames from the webcam and performs gesture recognition.The cap.read() function reads a frame from the webcam, and the return values ret and frame are assigned accordingly.The dimensions of the frame are extracted using the shape attribute of frame and assigned to the variables x, y, and c. x and y represent the width and height of the frame, respectively, and c represents the number of color channels.

Displaying the Output

The modified frame with visualized landmarks and gesture information is displayed in a separate window using cv2.imshow().The cv2.waitKey(1) function waits for a key press for 1 millisecond. If the pressed key is 'q', the program breaks out of the while loop.

Exiting the Program

Once the while loop is exited, the program releases the webcam resources using cap.release(). The windows displaying the frames are closed using cv2.destroyAllWindows().
