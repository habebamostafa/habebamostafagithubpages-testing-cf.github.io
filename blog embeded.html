Arduino


It is an open-source platform used for building electronics projects. It consists of both a physical programmable circuit board and a piece of software, or IDE that runs on your computer, used to write and upload computer code to the physical board.

The Arduino tool window consists of the toolbar with the buttons like verify, upload, new, open, save, serial monitor. pins we used in arduino

The pins on your Arduino are the places where you connect wires to construct a circuit probably in conjunction with a breadboard and some wire.

  • GND (3): Short for 'Ground'. There are several GND pins on the Arduino, any of which can be used to ground your circuit.

  • 5V (4) & 3.3V (5): As you might guess, the 5V pin supplies 5 volts of power, and the 3.3V pin supplies 3.3 volts of power. .

  • Analog (6): The area of pins under the 'Analog In' label (A0 through A5 on the UNO) are Analog In pins. These pins can read the signal from an analog sensor and convert it into a digital value that we can read.
  • Digital (7): Across from the analog pins are the digital pins (0 through 13 on the UNO). These pins can be used for both digital input and digital output .
 

  • PWM (8): You may have noticed the tilde (~) next to some of the digital pins (3, 5, 6, 9, 10, and 11 on the UNO). These pins act as normal digital pins but can also be used for something called (PWM). We have a tutorial on PWM, but for now, think of these pins as being able to simulate analog output .
  • AREF (9): Stands for Analog Reference. Most of the time you can leave this pin alone. It is sometimes used to set an external reference voltage (between 0 and 5 Volts) .

Connection

The Arduino project involves connecting various components to the Arduino board to create a system that controls a vehicle simulation. The project requires the use of an Arduino board, a LiquidCrystal LCD display, and several buttons. The connections are as follows: The steer input is connected to analog pin A2 on the Arduino board. The Faster button is connected to digital pin 11, while the Slower button is connected to digital pin 9. The Gear Up button is connected to digital pin 10, and the Gear Down button is connected to digital pin 12. The RS, EN, D4, D5, D6, and D7 pins of the LCD display are connected to digital pins 8, 7, 6, 5, 4, and 3 on the Arduino board, respectively. Finally, the Alarm output is connected to digital pin 2. By making these connections, the Arduino board can interface with the various components and control the vehicle simulation based on user input from the buttons and the steer input, displaying relevant information on the LCD display.

Software Processing

This code is a simplified simulation of a vehicle control system, where RPM, gear, steering input, and speed are monitored and controlled.

Libraries and Definitions:

The code begins by including the necessary libraries: Arduino.h and LiquidCrystal.h,It defines several constants using #define statements. These constants represent pin numbers, maximum values, and other parameters used in the code. 

Global Variables

Notable variables include RPM (representing the current RPM value), gear (representing the current gear), steerFactor (representing the steering factor input), and speed (representing the speed value).

LCD and Setup

The LiquidCrystal object lcd is created using the defined pin numbers for RS, EN, D4, D5, D6, and D7.The pin modes for the buttons and ALARM output are set using pinMode().

Main Loop

The main loop() function is where the core logic of the program resides.Depending on the button states (fast and slow) and the elapsed time since the last RPM change, the RPM value is incremented or decremented.

Gear Control

The code checks the button states and the current gear to control the gear value.If the Gear Up button is pressed and the conditions are met , the gear is incremented. The RPM value is adjusted accordingly.

Steering Control and LCD Display

If the steerFactor is within a certain range, it is considered as a neutral position.The LCD display is updated to show the gear value and an arrow indicating the steering direction.

Serial Communication and Speed

The speed value is converted from a string to an integer and adjusted accordingly. The alarm output pin is set high or low based on the received alarm value.

LCD Display of Speed

The LCD display is updated to show the current speed value.

Delay

The code includes a delay of 100 milliseconds to provide a time interval
